# Graph Explorations

Graph Explorations help you visualize relationships between intelligence objects so you can analyze threats, investigate cases, and share findings.

### Start a graph exploration <a href="#start-a-graph-exploration" id="start-a-graph-exploration"></a>

You can start a graph exploration in two ways:

#### From an intelligence object <a href="#from-an-intelligence-object" id="from-an-intelligence-object"></a>

Open any object page and click **New Graph Exploration**. This is the best starting point when you want to analyze a threat actor, malware family, technique, or other intelligence entity.

#### From a threat report <a href="#from-a-threat-report" id="from-a-threat-report"></a>

Start from an existing threat report graph when you want to reuse known relationships and expand the analysis from an existing context.

### Build the graph <a href="#build-the-graph" id="build-the-graph"></a>

#### Add relationships <a href="#add-relationships" id="add-relationships"></a>

Click an object in the graph to see related intelligence in the side panel. In the **Relationships** tab, you can review connected objects and add any relationship to the graph.

#### Remove elements <a href="#remove-elements" id="remove-elements"></a>

Keep the graph focused by removing elements you no longer need:

* Remove relationships from the relationship list.
* Remove objects by clicking the **trash icon** next to the object name.
* When you remove an object, related connections are removed **automatically.**

#### Use layers <a href="#use-layers" id="use-layers"></a>

Layers help you organize complex explorations into separate views. Each layer groups objects and relationships so you can keep different hypotheses or topics apart.

#### Default layer <a href="#default-layer" id="default-layer"></a>

Every new graph exploration starts with a default layer. It contains the initial objects and serves as the main workspace for the exploration.

#### Active layer <a href="#active-layer" id="active-layer"></a>

The layer panel is in the top-left corner of the graph. It shows the available layers, the active layer, and the layer where new objects and relationships will be added.

You can switch layers to move between different analytical perspectives.

### Save explorations <a href="#save-explorations" id="save-explorations"></a>

You can save graph explorations to preserve your work.

* New explorations need a name.
* Updates to existing explorations are saved directly.
* In the Graph Explorations page, changes are saved automatically.

Saved explorations can be reused later and shared with other team members
