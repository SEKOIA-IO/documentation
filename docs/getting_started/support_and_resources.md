# Support and resources

Sekoia offers several channels to help you get answers, learn new skills, and stay informed. This article lists all available resources and explains when to use each one.

## Get support

### Support portal

The Sekoia support portal is the primary channel for reporting bugs, asking technical questions, and opening tickets.

Access the support portal at [support.sekoia.com](https://support.sekoia.com).

Use the support portal when:

- You encounter a bug or unexpected platform behavior
- You need help with a configuration that is not covered in the documentation
- You need to escalate an issue that is affecting your operations

### Status page

To check whether Sekoia services are experiencing an incident or scheduled maintenance, visit the [Sekoia status page](https://status.sekoia.io).

The status page lists the operational health of all platform components by region. Subscribe to status updates to receive notifications about incidents and maintenance windows.

## Learn and train

### Sekoia Academy

The Sekoia Academy provides self-paced video courses and practical exercises for analysts and administrators.

Access the Academy at [academy.sekoia.com](https://academy.sekoia.com).

The Academy covers:

- Platform navigation and first steps
- Defend: intake configuration, rules activation, alert triage
- Intelligence: database navigation, report reading, feed configuration
- Advanced topics: playbook design, threat hunting, MITRE ATT&CK coverage analysis

Each quick start guide in this documentation links to the corresponding Academy module.

## Stay connected

### Community forum

The Sekoia Community is a GitHub-based space where customers and partners share knowledge, ask questions, and discuss use cases.

Join the community at [github.com/SEKOIA-IO/Community](https://github.com/SEKOIA-IO/Community).

### Changelog

The Sekoia changelog lists new features, improvements, and bug fixes across all products.

Access the changelog at [changelog.sekoia.com](https://changelog.sekoia.com).

Subscribe to the changelog to receive weekly or monthly summaries of platform updates.

## Contribute to the documentation

Sekoia's documentation is open source. If you find an error, a gap, or an opportunity to improve an article, you can submit a contribution.

To suggest a change:

1. Click **Edit this page** at the top of any documentation article.
2. Make your changes in the GitHub editor.
3. Submit a pull request with a description of your change.

The documentation team reviews all contributions and merges accepted changes.

The full documentation repository is available at [github.com/SEKOIA-IO/documentation](https://github.com/SEKOIA-IO/documentation).
