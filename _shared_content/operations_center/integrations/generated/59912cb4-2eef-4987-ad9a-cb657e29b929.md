
### Event Categories


The following table lists the data source offered by this integration.

| Data Source | Description                          |
| ----------- | ------------------------------------ |
| `Application logs` | LockSelf events are analyzed in detail |





In details, the following table denotes the type of events produced by this integration.

| Name | Values |
| ---- | ------ |
| Kind | `` |
| Category | `authentication`, `configuration`, `iam` |
| Type | `change`, `info`, `start` |




### Transformed Events Samples after Ingestion

This section demonstrates how the raw logs will be transformed by our parsers. It shows the extracted fields that will be available for use in the [built-in detection rules](/xdr/features/detect/rules_catalog.md) and hunting activities in the [events page](/xdr/features/investigate/events.md). Understanding these transformations is essential for analysts to create effective detection mechanisms with [custom detection rules](/xdr/features/detect/sigma.md) and to leverage the full potential of the collected data.

### Extracted Fields

The following table lists the fields that are extracted, normalized under the ECS format, analyzed and indexed by the parser. It should be noted that infered fields are not listed.

| Name | Type | Description                |
| ---- | ---- | ---------------------------|
|`@timestamp` | `date` | Date/time when the event originated. |
|`event.action` | `keyword` | The action captured by the event. |
|`event.category` | `keyword` | Event category. The second categorization field in the hierarchy. |
|`event.dataset` | `keyword` | Name of the dataset. |
|`event.outcome` | `keyword` | The outcome of the event. The lowest level categorization field in the hierarchy. |
|`event.type` | `keyword` | Event type. The third categorization field in the hierarchy. |
|`lockself.category.member.name` | `keyword` | The name of the category member. |
|`lockself.category.member.type` | `keyword` | The type of the category member. |
|`lockself.category.name` | `keyword` | The name of the category. |
|`lockself.category.permission.name` | `keyword` | The name of the category related permission. |
|`lockself.category.permission.status` | `keyword` | The status of the category related permission. |
|`lockself.connection.method` | `keyword` | The connection method name. |
|`lockself.connection.status` | `keyword` | The connection method status. |
|`lockself.credential.category.destination` | `keyword` | The destination category of the credential. |
|`lockself.credential.category.name` | `keyword` | The name of the credential category. |
|`lockself.credential.category.source` | `keyword` | The source category of the credential. |
|`lockself.credential.name` | `keyword` | The name of the credential. |
|`lockself.credential.sharing.object` | `keyword` | The number of shared credential(s). |
|`lockself.credential.tag.name` | `keyword` | The name of the credential tag. |
|`lockself.deposit_box.name` | `keyword` | The name of the deposit box. |
|`lockself.groups.names` | `keyword` | The name of the group(s). |
|`lockself.option.scope` | `keyword` | The scope of the option. |
|`lockself.option.type` | `keyword` | The type of the option. |
|`lockself.option.value` | `keyword` | The new value of the option. |
|`lockself.transfer.file.name` | `keyword` | The transfer file name. |
|`lockself.user.status.current` | `keyword` | The current status of the user. |
|`lockself.user.status.previous` | `keyword` | The previous status of the user. |
|`observer.product` | `keyword` | The product name of the observer. |
|`observer.vendor` | `keyword` | Vendor name of the observer. |
|`organization.name` | `keyword` | Organization name. |
|`source.ip` | `ip` | IP address of the source. |
|`source.user.email` | `keyword` | User email address. |
|`user.target.email` | `keyword` | User email address. |
|`user_agent.name` | `keyword` | Name of the user agent. |
